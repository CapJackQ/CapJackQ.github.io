Objective-C Methods

发表于 2016-06-17 | 分类于 iOS

生活壞到一定程度就會好起來，因為它無法更壞。努力過後，才知道許多事情，堅持堅持，就過來了

Methods

方法代表了对象知道该如何去执行的行为. 方法与属性是逻辑对应的, 代表了一个对象的数据. 你可以认为方法就是函数, 它附属与对象; 但是它们之间却有着不同的语法.

在本部分, 我们将探究Objective-C的方法命名习惯. 它的命名习惯会使来自C++, Java, Python和类似语言的有经验的开发者们心烦. 同时会简短地讨论Objective-C的访问修饰符(或者lack thereof), 以及学习如何参考使用方法选择器.

Naming Conventions

Objective-C方法被设计用来去除API中所有模棱两可之处. 其结果是, 方法名极度的冗长,啰嗦的叙述性语句. 最终归纳起来Objective-C的方法命名有三个简单地规则:

  1. 不要缩写任何东西
  2. 明确规定方法本身的参数名
  3. 明确描述方法的返回值

Abbreviations

使方法最容易理解和可预计的途径是避免简单地缩写. 大部分Objective-C程序员_预计_方法要书写完整, 这也是所有从Foundation到UIKit标准框架的习惯. 这就是为什么上面接口中选择maximumSpeed而不是maxSpeed.

Parameters

Objective-C的冗长设计理念最清晰的例子之一是为方法参数命名的习惯. 然而C++, Java和其他Simula-style语言将方法看做是与其参数独立开来的实体, 但是, Objective-C方法命名实际上包含了其所有的参数名.

比如, C++中要使汽车转弯90度, 你会调用像类似于turn(90)的方法. 然而, Objective-C中这样做是有歧义的. 因为方法turn()的参数的类型是不明确的-有可能是新的行驶方向, 或者是在当前行驶方向上增加的角度值. Objective-C中通过使用前置词描述参数来明确其含义. 由此产生的API确保方法永远不会错误的阐述: 是turnByAngle:90或者turnToAngle:90.

当一个方法接受了多个参数时, 每个参数名也是包含在方法名中. 例如, 上面的initWithModel:mileage:方法明确地标明了Model和mileage. 等会我们会看到, 这种方法提供了详尽的方法调用信息.

Return Values

同样你会注意到任何方法返回值都明确描述了其返回值是什么. 有时这样做同陈述返回类型的类同样简单, 但有时你必须为其添加可描述形容词前缀.

比如, 以factory methods开始的car, 其明确描述了方法将返回一个Car类的实例. fasterCar:和slowerCar:两个类比方法, 返回接收器和参数的faster/lower, 而且这也同样在API中又明确表达. 值得注意的是, 单例方法也应该遵从这种模式(e.g., shareCar), 因为常规的实例方法名称不明确.

更多的命名习惯请移步官方: Cocoa Coding Guidlines.

Protected and Private Methods

Objective-C中方法并没有protected或者private访问修饰词—方法都是公共的, public. 但是, Objective-C提供了可供选择的结构范例, 可以让你模仿这些特性.

“private”方法可以在类的实现文件中创建, 同时在其接口文件中不用写出. 由于其他对象(包括子类)不会引入实现文件, 这些方法实际上对其他类来说是隐藏的.

代替受保护的方法, Objective-C提供了类目(categories)来实现, 它是作为API隔离部分常用的解决办法. 完整的例子请移步“Protected” Methods.

Selectors

方法选择器(selector)是Objective-C方法名的内部表示方法. selectors让你将方法看做是一个独立的实体, 能够将需要实现的行为和对象分开. 这是Target-Action模式的依据.

有两种方法可以获取一个方法名的方法选择器. @selector()指令使你转换源代码方法名为一个方法选择器, NSSelectorFromSting()函数让你将字符串转换为一个方法选择器(后者效率低). 两者都返回一个特殊的返回值类型称为SEL. 你可以同BOOL, int或其他数据类型一样使用SEL.

方法选择器能通过performSelector:和相关方法在任意对象上执行. withObject:为方法传递一个参数(或两个), 但要求这些参数全都是对象(objects). 如果这过于限制您的需求, 请移步官网查看NSInvocation高级用法. 如果你不确定目标对象是否定义了这个方法, 你应当在尝试执行方法选择器之前使用respondsToSelector:来检查以下.

一个方法的专业名称是将主要方法名和所有参数标签结合起来, 用冒号隔开. 这样就会使冒号成为了方法名的一部分, 而容易让Objective-C的初学者搞混. 它们的用法可以总结如下: _无参方法不包含冒号, 带参方法总是以冒号结束_.
