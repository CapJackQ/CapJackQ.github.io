Objective-C学习Functions

发表于 2016-05-11 | 分类于 iOS

本文翻译自Ry’s Objective-C Tutorial

Functions

同变量(variables), 条件语句(conditionals)和循环(loops), 函数(functions)一起构成了现代编程语言的基础. 它们允许你重用apps中的任意代码块(arbitrary block of code), 而这些代码块是必须用来组织和维护所有最琐碎的代码库. 在iOS和OS X框架中会发现很多这样的例子
和它的其他结构, Objective-C的函数完全依赖于C编程语言. 本部分介绍C函数最重要的方面, 包括语法, 独立的定义和实现, 普通的作用域问题以及函数库.

Basic Syntax

C语言中函数有四个组成: 返回值(return value), 函数名(name), 参数(parameters)和功能代码部分(code block).

Declarations vs. Implementations

函数须在其调用之前定义好. 如果将上例中getRandomMake()函数在main()函数之后定义, 那么在主函数中调用该函数的时候编译器将无法找到函数. 这种对开发者而言是相当精确的结构使得组织大的apps变得困难. 为了解决这一问题, C语言允许将函数定义从其实现中分离出来.

函数的定义和实现

函数的定义告诉编译器该函数的输入(intputs)和输出(outputs)是怎样的. 通过所提供的函数返回值和参数类型, 编译器可以确认你在正确的使用函数而不用知晓它实际上在做什么. 函数的具体实现则是添加其功能代码块. 两者一起就完整定义了一个函数.

The Static Keyword

静态(static)关键词能够改变函数或者变量的可用性(availability). 不幸地是, 这种影响取决于你在哪使用它(static keyword).

Static Functions

一般来说, 所有的函数都有全局作用域. 这意味着当你在一个文件中定义了一个函数时, 它在任其他何一处都可以使用. static 关键词将函数的作用范围限制在了当前文件内, 这非常有利于创建私有(“private”)函数以及避免命名冲突的问题.

Static Local Variable

声明在函数内部的变量(也称为自动局部变量(automatic local variables))会在函数被调用时重新设置. 这是一种直观地默认行为(intuitive default behavior), 作为函数的一贯表现不论调用多少次都是如此. 但是如果使用static修饰局部变量, 那么函数就会通过声明而察觉到.

Objective-C不支持命名空间, 因此为了不引起与其他全局函数命名冲突, 大的框架需要函数(和类)的前缀来做唯一标识符. 这就是为什么创建函数时会看到是NSMakeRange()和CGImageCreate()来代替makeRange()和imageCreate().
