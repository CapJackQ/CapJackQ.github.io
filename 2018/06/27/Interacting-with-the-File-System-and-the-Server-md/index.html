上传文件

与使用HTTP协议将文件从服务器加载到浏览器相反，PHP则是将文件从浏览器上传至服务器。通常使用HTML表格交互来实现这一结构。

使用一个简单的界面

File Upload Configuration Setting in php.ini

指令 描述 缺省值
file_uploads 控制是否允许上传HTTP文件。值为On/Off On
upload_tmp_dir 指示当被上传文件等待处理时的暂时存储路径。若没有设置，系统使用缺省值(如/tmp) NULL
upload_max_filesize 控制上传文件的最大允许大小。如果大于该值，PHP将会写入0字节的占位文件替代。内容大小可以使用简写格式：K表示千字节，M表示兆字节，G表示千兆字节。 2M
post_max_size 控制PHP可接受的最大POST数据大小。该数据必须大于upload_max_filesize指令的数据，因为它是所有post数据大小，包括所有上传的文件。可以使用同上个指令相同的简写。 8M

上传文件的HTML表单

上传文件只能使用POST。

enctype默认值为application/x-www-form-urlencoded，在发送前对所有字符进行编码（将空格转换为 “+” 符号，特殊字符转换为 ASCII HEX 值）；multipart/form-data，不对字符编码。当使用有文件上传控件的表单时，该值是必需的；text/plain，将空格转换为 “+” 符号，但不编码特殊字符。

处理文件

当文件上传后，会暂时进入临时目录，这个目录是在php.ini中的upload_tmp_dir指令表明。如果没有设置，则会默认为web服务器的主临时目录，比如/tmp。如果在脚本执行完之前没有移动，拷贝或重命名文件，则会在脚本结束后删除改文件。

PHP脚本中需要处理的数据存储在超全局数组$_FILES中。$_FILES的入口将被存储在HTML表单中标签名为<file>的标签。表单元素命名为the_file，因此数组会有以下内容

  • 存入$_FILES['the_file']['tmp_name']的值是已经临时存储在web服务器上的临时名和位置
  • 存入$_FILES['the_file']['name']的值是上传文件的原始名
  • 存入$_FILES['the_file']['size']的值是上传文件的字节大小
  • 存入$_FILES['the_file']['type']的值是文件MIME类型，比如text/plain或image/png
  • 存入$_FILES['the_file']['error']的值给出错误码

给出文件的路径和文件名，就可以将其复制到有用的地方。在脚本执行完之后会删掉临时文件。

错误信息$_FILES['userfile']['error']参见http://php.net/manual/zh/features.file-upload.errors.php。

同时会检查MIME类型确保上传制定类型的文件。示例中之上传PNG文件。因此需要将$_FILES['the_file']['tmp_name']值设置为image/png。这个仅仅是错误检查而非安全检查。MIME类型由用户的浏览器用文件的扩展名和内容解释，然后传回服务器。
