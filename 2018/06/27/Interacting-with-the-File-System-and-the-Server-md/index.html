上传文件

与使用HTTP协议将文件从服务器加载到浏览器相反，PHP则是将文件从浏览器上传至服务器。通常使用HTML表格交互来实现这一结构。

使用一个简单的界面

File Upload Configuration Setting in php.ini

指令 描述 缺省值
file_uploads 控制是否允许上传HTTP文件。值为On/Off On
upload_tmp_dir 指示当被上传文件等待处理时的暂时存储路径。若没有设置，系统使用缺省值(如/tmp) NULL
upload_max_filesize 控制上传文件的最大允许大小。如果大于该值，PHP将会写入0字节的占位文件替代。内容大小可以使用简写格式：K表示千字节，M表示兆字节，G表示千兆字节。 2M
post_max_size 控制PHP可接受的最大POST数据大小。该数据必须大于upload_max_filesize指令的数据，因为它是所有post数据大小，包括所有上传的文件。可以使用同上个指令相同的简写。 8M

上传文件的HTML表单

上传文件只能使用POST。

enctype默认值为application/x-www-form-urlencoded，在发送前对所有字符进行编码（将空格转换为 “+” 符号，特殊字符转换为 ASCII HEX 值）；multipart/form-data，不对字符编码。当使用有文件上传控件的表单时，该值是必需的；text/plain，将空格转换为 “+” 符号，但不编码特殊字符。

处理文件

当文件上传后，会暂时进入临时目录，这个目录是在php.ini中的upload_tmp_dir指令表明。如果没有设置，则会默认为web服务器的主临时目录，比如/tmp。如果在脚本执行完之前没有移动，拷贝或重命名文件，则会在脚本结束后删除改文件。

PHP脚本中需要处理的数据存储在超全局数组$_FILES中。$_FILES的入口将被存储在HTML表单中标签名为<file>的标签。表单元素命名为the_file，因此数组会有以下内容

  • 存入$_FILES['the_file']['tmp_name']的值是已经临时存储在web服务器上的临时名和位置
  • 存入$_FILES['the_file']['name']的值是上传文件的原始名
  • 存入$_FILES['the_file']['size']的值是上传文件的字节大小
  • 存入$_FILES['the_file']['type']的值是文件MIME类型，比如text/plain或image/png
  • 存入$_FILES['the_file']['error']的值给出错误码

给出文件的路径和文件名，就可以将其复制到有用的地方。在脚本执行完之后会删掉临时文件。

Session Upload Progress

在5.4版本，PHP包含一个选项可以追踪上传文件的过程。该选项对使用AJAX为用户返回实时信息的网页应用特别有用。在以前上传文件的模式中，无法通知用户关于其上传过程的信息，直到上传过程结束。但若使用PHPsession upload progress功能，就可以收到正在上传时的信息并将其展示给用户。

在使用session upload progress之前确保以下指令是可用的。指令可在php.ini中找到。(/usr/local/etc/php/7.0)

Directive Description Default Value
session.upload_progress.enabled 可以使用$_SESSION全局变量来跟踪上传 On
session.upload_progress.cleanup 在POST数据完全上传之后清理信息 On
session.upload_progress.prefix 前缀作为$_SESSION中的一部分上传用于唯一标识 upload_progress
session.upload_progress.name $_SESSION中的key PHP_SESSION_UPLOAD_PROGRESS
session.upload_progress.freq 定义session upload progress信息更新频率 1%
session.upload_progress.min_freq 定义信息更新的最小延迟，单位是s 1

当这些配置完成后，$_SESSION全局变量会包含将要上传文件的信息。如果打印所有的变量，它们会包含在$_SESSION中。

使用程序执行函数

  • exec()：string exec(string command [, array &result [, int &return_value]])，传入在终端输入的指令，比如exec("ls -la")；exec()没有直接输出，返回结果的最后一行。

    如果传入一个变量作为result，就会得到一个关于该条命令的数组结果。如果传入变量作为result_value，就会得到返回代码。

  • passthru()：void passthru(string command [, int return_value])。该函数直接将结果输出到浏览器。参数同上

  • system()：string system(string command [, int return_value])。该函数输出结果到浏览器。
