Protocols

协议是能够被_任何_一个类所实现的一组相关的属性和方法. 它们能让你在完全不相干的类当中复用单一的API声明, 所以它们比任何一个类的接口更加灵活. 这种特性使它在现有的类层次结构之上表现水平关系成为可能.

Methods

方法代表了对象知道该如何去执行的行为. 方法与属性是逻辑对应的, 代表了一个对象的数据. 你可以认为方法就是函数, 它附属与对象; 但是它们之间却有着不同的语法.

在本部分, 我们将探究Objective-C的方法命名习惯. 它的命名习惯会使来自C++, Java, Python和类似语言的有经验的开发者们心烦. 同时会简短地讨论Objective-C的访问修饰符(或者lack thereof), 以及学习如何参考使用方法选择器.

Properties

Objective-C Classes

发表于 2016-05-17

Classes

同其它面向对象编程语言一样, Objective-C的类为创建对象提供了蓝图. 首先, 在一个类中定义了属性(properties)和行为(behaviors)的集合. 然后, 从该类中实例化对象与属性和行为交互.

Objective-C同C++类似, 将类的接口从实现中抽离出来. 接口(interface)声明了类的公用的属性和方法, 与之对应的实现(implementation)则定义了这些属性和方法的实现代码. 这与之前提到的函数的定义与实现分离是相同的.
